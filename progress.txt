# 🧱 Proyecto

Sistema de gestión de pedidos basado en microservicios:

- API Gateway
- users-service
- products-service
- orders-service

---

# 🛠️ Tecnologías

- Node.js
- TypeScript
- Express
- ES Modules (type: module)
- tsx (dev runtime)
- Kafka (KafkaJS)
- Docker & Docker Compose
- Axios (comunicación HTTP entre servicios)
- UUID

---

# 🧠 Arquitectura objetivo

- Cada microservicio es completamente independiente.
- El API Gateway es el único punto de entrada externo.
- Comunicación:
  - HTTP para consultas síncronas
  - Kafka para eventos desacoplados
- Estructura por capas en cada servicio:
  - routes → controllers → services
- Se introducirán progresivamente patrones como:
  - DTOs
  - Middleware
  - Error handling centralizado
  - Repository pattern
  - Event-driven architecture con Kafka

## Funcionalidad actual del users-service

### Endpoints existentes:

- GET /health → health check del servicio
- POST /users → crear usuario
- GET /users → obtener todos los usuarios
- GET /users/:id → obtener usuario por ID

---

## Modelo de dominio

```ts
User {
  id: string;

  email: string;

CreateUserDto {
```

---

## Estado de la lógica

- Usuarios almacenados en memoria (users: User[])
- UUID usado para generación de IDs
- Validación básica implementada con middleware:
  - validateCreateUser
- Flujo actual:
  request → middleware → controller → service → response

---

## Buenas prácticas ya aplicadas

- Separación estricta de capas
- Services sin conocimiento de HTTP
- Controllers como capa de adaptación HTTP
- Middleware para validación básica
- Uso inicial de TypeScript para contratos
