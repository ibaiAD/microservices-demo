Proyecto:
Un sistema sencillo de gestión de pedidos compuesto por:

* API Gateway
* users-service
* products-service
* orders-service

Tecnologías:

* Node.js
* TypeScript
* Express
* ES Modules (type: module)
* tsx como entorno de desarrollo
* Kafka (KafkaJS)
* Docker y Docker Compose
* Axios para comunicación HTTP
* UUID

Arquitectura:

* Cada microservicio será independiente.
* El Gateway será el único punto de entrada.
* Los servicios se comunicarán mediante HTTP cuando tenga sentido y mediante eventos de Kafka cuando queramos desacoplar procesos.
* Seguiremos una estructura profesional (controllers, services, routes, etc.) y explicaremos cuándo introducir patrones como Repository, DTOs o middleware.
