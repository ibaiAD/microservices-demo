MANEJO DE ERRORES IMPLEMENTADO

Se ha introducido un sistema básico de errores personalizados.

Estructura:

src/errors/

AppError.ts
NotFoundError.ts

Concepto:

- AppError hereda de Error.
- Tiene:
  - message
  - statusCode

- NotFoundError hereda de AppError.
- Usa statusCode 404.

Ejemplo:

throw new NotFoundError("User not found")

ERROR HANDLER GLOBAL

Existe:

src/middlewares/errorHandler.ts

Está registrado al final de app.ts.

Responsabilidad:

- Si recibe un AppError:
  - devuelve su statusCode.
  - devuelve su message.

- Si recibe un error desconocido:
  - devuelve 500.

Flujo:

throw AppError
      ↓
errorHandler
      ↓
HTTP response

CAMBIO IMPORTANTE REALIZADO

Antes:

El controller decidía errores:

if (!user) {
  return res.status(404)
}

Ahora:

El service decide:

export function getUserById(id: string): User {
  const user = users.find(user => user.id === id)

  if (!user) {
    throw new NotFoundError("User not found")
  }

  return user
}

Los controllers ya no conocen reglas de negocio.

ASYNC ERROR HANDLING INTRODUCIDO

Se creó:

src/utils/asyncHandler.ts

Implementación actual:

import type { RequestHandler } from "express";

export function asyncHandler(fn: RequestHandler): RequestHandler {
  return (req, res, next) => {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
}

Motivo:

Preparar Express para controllers async y errores provenientes de Promises.

ESTADO ACTUAL DE RUTAS

users.routes.ts:

router.post(
  '/users',
  validateCreateUser,
  asyncHandler(createNewUser)
)

router.get(
  '/users',
  asyncHandler(getAllUsers)
)

router.get(
  '/users/:id',
  asyncHandler(getUser)
)

Decisión tomada:

- Envolver controllers desde las rutas.
- No envolver los controllers internamente.

Motivo:

La adaptación a Express debe quedarse en la capa HTTP.

ESTADO ACTUAL DE CONTROLLERS

Ejemplo:

export async function getUser(
  req: Request,
  res: Response
) {

Actualmente son async, aunque todavía no usan await.

La intención es prepararlos para cuando los services sean realmente asíncronos.
